Reaction Time Game – ATmega8 Embedded System

Built a reaction time measurement system around the ATmega8 that integrates analog input, digital I/O, timers, interrupts, PWM, EEPROM, I²C communication, and UART debugging into one embedded application.

Project Details

ATmega8Embedded CADCPWMEEPROMI²C

System Architecture

The system uses six gameplay push buttons, one start/stop button through an external interrupt, and a potentiometer connected to the ADC for dynamic difficulty control. An ATmega8 handles processing, with Timer0 measuring reaction time and Timer1 generating PWM for a servo indicator. EEPROM stores high-score data across power cycles. Outputs include six LEDs, a 16×2 I²C LCD showing score and settings, and a servo motor indicating difficulty.

Technical Decisions

Hardware timers were used instead of delay-based timing for accurate reaction measurement. State machine logic prevented invalid transitions during gameplay. ADC input was used to change difficulty dynamically. PWM via Timer1 controlled the servo, and EEPROM storage preserved the high score after power loss. Software debouncing reduced false button triggers.

Challenges

Challenges included software I²C timing issues affecting LCD reliability, blocking delays interfering with real-time behavior, button bounce causing false triggers, invalid input detection, and state conflicts during rapid user interaction.

Results

The final system achieved accurate millisecond-level reaction timing, dynamic difficulty control, persistent high-score storage, and successful integration of ADC, timers, interrupts, PWM, I²C, UART, and EEPROM into one stable embedded application.

Highlights

  • Measures reaction time between LED stimulus and user response.
  • Uses hardware timers for accuracy and PWM for servo indication.
  • Stores high score in EEPROM and adjusts difficulty using a potentiometer.
  • Demonstrates stable real-time behavior using direct register-level programming.
